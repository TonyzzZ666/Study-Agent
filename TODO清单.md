# 学习养成计划 — 一周实践 TODO

> **课程**：软件系统实践（小学期） | **2人** | **周期：7天**
> **核心目标**：跑通 注册→建任务→打卡→看统计 这条链路，能完整演示即可

---

## 一周时间规划

```
周一            周二            周三           周四            周五          周六          周日
环境搭建   →   后端开发    →   前端开发   →   联调测试   →   文档报告   →   收尾   →   答辩
```

| 天数 | 阶段 | 目标 |
|------|------|------|
| Day 1 | 🔧 环境搭建 | 项目跑起来、数据库建好 |
| Day 2 | ⚙️ 后端 | 所有 API 写完并能用 Postman 调通 |
| Day 3 | 🖥️ 前端 | 页面写完、能调后端 |
| Day 4 | 🔗 联调 | 前后端打通、全流程走通 |

## 砍掉的东西（人要有自知之明）

| 砍掉 | 原因 |
|------|------|
| ❌ 推荐任务模板 | 预置数据 + 独立页面，工作量大但加分少 |
| ❌ 提醒系统 | 要定时任务 + 消息表 + 前端消息中心，至少多半天 |
| ❌ ECharts 图表 | Element Plus 进度条够用了，别折腾 |
| ❌ 打卡日历热力图 | 好看但费时间 |
| ❌ 个人资料编辑 | 不是核心流程，先砍 |
| ❌ 任务搜索 | 2人项目数据量小，筛选就够了 |

## 数据库（3张表）

| 表 | 字段 |
|----|------|
| user | id, username, password, nickname, created_at |
| task | id, user_id, title, description, priority(HIGH/MEDIUM/LOW), deadline, status(TODO/DONE), created_at |
| check_in | id, user_id, task_id, check_time |

---

## API 清单（8个）

| 方法 | 路径 | 说明 | JWT |
|------|------|------|-----|
| POST | /api/user/register | 注册 | ❌ |
| POST | /api/user/login | 登录，返回Token | ❌ |
| GET | /api/tasks | 任务列表 | ✅ |
| POST | /api/tasks | 创建任务 | ✅ |
| PUT | /api/tasks/{id} | 编辑任务 | ✅ |
| DELETE | /api/tasks/{id} | 删除任务 | ✅ |
| POST | /api/checkin/{taskId} | 打卡 | ✅ |
| GET | /api/statistics | 完成率+打卡天数 | ✅ |
